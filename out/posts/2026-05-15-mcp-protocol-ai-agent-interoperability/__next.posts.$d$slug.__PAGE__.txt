3:T74ba,<h2 class="text-2xl font-black text-[var(--color-text)] mt-12 mb-4 tracking-tight border-b border-[var(--color-border)] pb-3">前言：Agent孤岛困境</h2><p class="text-[var(--color-text-2)] leading-relaxed my-4">过去两年，AI Agent 生态有一个显著的痛点：每个平台、每个框架、每个模型供应商都有一套自己的 Agent 通信协议。</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">OpenAI 的 Agents SDK、Anthropic 的 Claude Agent Anthropic 在 2025 年初提出，目标是成为 AI Agent 与外部世界交互的通用接口层——就像 USB-C 之于设备连接，让不同厂商的设备终于可以用同一套物理接口互联。</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">本文从协议设计角度，深度拆解 MCP 的架构、核心概念、以及工程实践。</p><h2 border-[var(--color-border)]'>`</code></p><h2 class="text-2xl font-black text-[var(--color-text)] mt-12 mb-4 tracking-tight border-b border-[var(--color-border)] pb-3">三、采样机制：让模型"主动"使用工具</h2><p class="text-[var(--color-text-2)] leading-relaxed my-4">MCP 有一个独特的机制：<strong class='text-[var(--color-text)] font-semibold'>Sampling</strong>。</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">在标准协议中，Client 负责调用工具，Host 决定何时调用。但 Sampling 允许 Server 主动请求模型生成内容——这解决了"Agent 想生成子任务"的问题，比如：</p><ul class='space-y-2 my-5 ml-4 list-disc'><li class="text-[var(--color-text-2)] leading-relaxed">Agent 想生成一个子 Agent 来处理某个步骤</li><li class="text-[var(--color-text-2)] leading-relaxed">Agent 需要模型对某个中间结果做"再理解"</li><li class="text-[var(--color-text-2)] leading-relaxed">多 Agent 协作时，子 Agent 需要动态请求主模型的决策</li></ul><p class="text-[var(--color-text-2)] leading-relaxed my-4">采样请求通过 <code class='bg-[var(--color-surface)] text-[var(--color-accent)] px-1.5 py-0.5 rounded text-sm font-mono border border-[var(--color-border)]'> SamplingMessage </code> 类型传递，Server 发请求给 Host，Host 转发给模型，结果返回给 Server。</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">这个机制在多智能体编排场景下特别有价值。</p><h2 class="text-2xl font-black text-[var(--color-text)] mt-12 mb-4 tracking-tight border-b border-[var(--color-border)] pb-3">四、安全模型：MCP的安全边界</h2><p class="text-[var(--color-text-2)] leading-relaxed
