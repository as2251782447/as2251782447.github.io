3:T8a6b,<h2 class="text-2xl font-black text-[var(--color-text)] mt-12 mb-4 tracking-tight border-b border-[var(--color-border)] pb-3">背景：为什么 LLM 推理这么难优化</h2><p class="text-[var(--color-text-2)] leading-relaxed my-4">大语言模型推理和传统深度学习推理有个本质区别：<strong class='text-[var(--color-text)] font-semibold'>输入输出长度不固定</strong>。一个请求可能只输出 20 个 token，另一个请求要输出 2000 个 token。如果简单按请求排队，等短请求的人被长请求堵死，GPU class='text-[var(--color-text)] font-semibold'>Continuous Batching</strong>，GPU 利用率直接上一个台阶。</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">今天我们来深度拆解这套机制，配合 PyTorch 代码示例，搞清楚它为什么有效。</p><h2 class="text-2xl font-black text-[var(--color-text)] mt-12 mb-4 tracking-tight border-b border-[var(--color-border)] pb-3">1. 经典 Static Batching 的瓶颈</h2><p class="text-[var(--color-text-2)] leading-relaxed my-4">先看静态分批的问题。假设我们有 4 个请求，长度各不相同：</p><p class="text-[var(--color-text-2)] leading-relaxed my-4"><code class='bg-[var(--color-surface)] text-[var(--color-accent)] px-1.5 py-0.5 rounded text-sm font-mono border border-[var(--color-border)]'>`</code></p><p class="text-[var(--color-text-2)] leading-relaxed my-4">Request A: 512 input  +  64 output</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">Request B: 512 input  + 128 output</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">Request C: 1024 input +  32 output</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">Request D: 1024 input + 256 output</p><p class="text-[var(--color-text-2)] leading-relaxed my-4"><code class='bg-[var(--color-surface)] text-[var(--color-accent)] px-1.5 py-0.5 rounded text-sm font-mono border border-[var(--color-border)]'>`</code></p><p class="text-[var(--color-text-2)] leading-relaxed my-4">静态分批要把它们凑成一批处理，必须 padding 到最大长度：</p><p class="text-[var(--color-text-2)] leading-relaxed my-4"><code class='bg-[var(--color-surface)] text-[var(--color-accent)] px-1.5 py-0.5 rounded text-sm font-mono border border-[var(--color-border)]'>`</code></p><p class="text-[var(--color-text-2)] leading-relaxed my-4">Batch: 1024 (max) input + 256 (max) output = 1280 tokens per iteration</p><p class="text-[var(--color-text-2)] leading-relaxed my-4"><code class='bg-[var(--color-surface)] text-[var(--color-accent)] px-1.5 py-0.5 rounded text-sm font-mono border border-[var(--color-border)]'>`</code></p><p class="text-[var(--color-text-2)] leading-relaxed my-4">每个请求实际有用的 token 比例很低，大部分计算是无效的 padding。更糟糕的是，Request A 在 64 步后就可以返回了，但它必须等 Batch 里最慢的 D 跑完 256 步。这是 <strong class='text-[var(--color-text)] font-semibold'>死等问题（Head-of-Line Blocking）</strong>。</p><p class="text-[var(--color-text-2)] leading-relaxed my-4"><code class='bg-[var(--color-surface)] text-[var(--color-accent)] px-1.5 py-0.5 rounded text-sm font-mono border border-[var(--color-border)]'>`</code></p><p class="text-[var(--color-text-2)] leading-relaxed my-4">Timeline (iteration steps):</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">Step 1-64:   [A B C D]  四路并行</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">Step 65-128: [  B C D ]  A 结束但占着位置</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">Step 129-256:[    C D  ] B 结束但占着位置</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">Step 257-320:[      D  ] 只有 D 在跑</p><p class="text-[var(--color-text-2)] leading-relaxed my-4"><code class='bg-[var(--color-surface)] text-[var(--color-accent)] px-1.5 py-0.5 rounded text-sm font-mono border border-[var(--color-border)]'>`</code></p><p class="text-[var(--color-text-2)] leading-relaxed my-4"><strong class='text-[var(--color-text)] font-semibold'>GPU 利用率曲线就是一个波峰然后漫长的低谷。</strong></p><h2 class="text-2xl font-black text-[var(--color-text)] mt-12 mb-4 tracking-tight border-b border-[var(--color-border)] pb-3">2. Continuous Batching：迭代级调度</h2><p class="text-[var(--color-text-2)] leading-relaxed my-4">Continuous Batching 的核心思想：<strong class='text-[var(--color-text)] my-4">这是迭代级调度，不是请求级调度。调度发生在每个生成步，而不是每个请求结束时。</p><h3 class="text-base font-bold text-[var(--color-text)] mt-8 mb-3">2.1 调度循环</h3><p class="text-[var(--color-text-2)] leading-relaxed my-4"><code class='bg-[var(--color-surface)] text-[var(--color-accent)] px-1.5 py-0.5 rounded text-sm font-mono border border-[var(--color-border)]'>`</code>python</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">import torch</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">from typing import List, Dict</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">class class="text-[var(--color-text-2)] leading-relaxed my-4">"""</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">def __init__(self, max_batch_size: int, device: str = "cuda"):</p><p class="text-[var(--color-text-2)] leading-relaxed my-4"># running 表示正在生成的请求</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">self.running: List[GenerationRequest] = []</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">def step(self,</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">requests: List[GenerationRequest],</p><p text-[var(--color-accent)] px-1.5 py-0.5 rounded text-sm font-mono border border-[var(--color-border)]'>`</code></p><p class="text-[var(--color-text-2)] leading-relaxed my-4">Static Batching:</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">[======ABCD======]              [==EFGH==]</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">busy       idle gap             busy</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">Continuous Batching:</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">[==A==]                          [==E==]</p><p [==GH==]</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">[==D==]   [==D===]    [==D====]</p><p class="text-[var(--color-text-2)] leading-relaxed my-4"><code class='bg-[var(--color-surface)] text-[var(--color-accent)] px-1.5 py-0.5 rounded text-sm font-mono border border-[var(--color-border)]'>`</code></p><p class="text-[var(--color-text-2)] leading-relaxed my-4">从四路并行逐渐变成一路，然后立刻有新请求填进来。没有长空闲。</p><h2 class="text-2xl font-black text-[var(--color-text)] mt-12 mb-4 tracking-tight border-b border-[var(--color-border)] pb-3">3. GPU 显存根本装不下。</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">vLLM 提出的 <strong class='text-[var(--color-text)] font-semibold'>PagedAttention</strong> 灵感来自操作系统的分页管理：不再一次性分配一大块连续显存给 KV class="text-[var(--color-text-2)] leading-relaxed my-4">num_free_slots: int = 16</p><p class="text-[var(--color-text-2)] leading-relaxed my-4"># 物理显存指针</p><p class="text-[var(--color-text-2)] set(range(total_blocks))</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">def allocate(self, num_tokens: int) -> List[int]:</p><p class="text-[var(--color-text-2)] leading-relaxed class="text-[var(--color-text-2)] leading-relaxed my-4">self.num_generated: int = 0</p><p class="text-[var(--color-text-2)] leading-relaxed my-4"><code class='bg-[var(--color-surface)] text-[var(--color-accent)] px-1.5 py-0.5 rounded text-sm font-mono border border-[var(--color-border)]'>`</code></p><h3 class="text-base font-bold text-[var(--color-text)] mt-8 mb-3">3.2
