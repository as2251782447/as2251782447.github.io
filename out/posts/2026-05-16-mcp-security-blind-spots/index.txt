10:T4fb0,<p class="text-[var(--color-text-2)] leading-relaxed my-4"># MCP 协议的安全盲区：你的 AI 助手正在访问什么</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">2025 年底，Model Context Protocol（MCP）从 Anthropic 的内部实验变成开源协议后，迅速被采纳。Cursor、Claude Desktop、 Zed、Cloudflare Workers AI 等主流工具纷纷支持 MCP，一时间「所有 AI 工具都能调用你的数据库、文件系统、Slack、GitHub」成了标配能力。</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">但问题来了：<strong class='text-[var(--color-text)] font-semibold'>当协议层面缺乏安全边界时，这个能力就成了攻击面。</strong></p><p class="text-[var(--color-text-2)] leading-relaxed my-4">本文从实际漏洞案例出发，系统梳理 MCP 协议的安全盲区，以及如何在生产环境中构建防御。</p><h2 class="text-2xl font-black text-[var(--color-text)] mt-12 mb-4 tracking-tight border-b border-[var(--color-border)] pb-3">什么是 MCP——30 秒背景</h2><p class="text-[var(--color-text-2)] leading-relaxed my-4">MCP 的核心是一个双向 JSON-RPC 协议：</p><p class="text-[var(--color-text-2)] leading-relaxed my-4"><code class='bg-[var(--color-surface)] text-[var(--color-accent)] px-1.5 py-0.5 rounded text-sm font-mono border border-[var(--color-border)]'>`</code></p><p class="text-[var(--color-text-2)] leading-relaxed my-4">Client (AI App)  ←→  MCP Server (工具/数据源)</p><p class="text-[var(--color-text-2)] leading-relaxed my-4"><code class='bg-[var(--color-surface)] text-[var(--color-accent)] px-1.5 py-0.5 rounded text-sm font-mono border border-[var(--color-border)]'>`</code></p><p class="text-[var(--color-text-2)] leading-relaxed my-4">MCP Server 暴露 <strong class='text-[var(--color-text)] font-semibold'>Resources</strong>（数据）、<strong class='text-[var(--color-text)] font-semibold'>Tools</strong>（可执行操作）、<strong class='text-[var(--color-text)] font-semibold'>Prompts</strong>（模板）。Client 在运行时动态发现并调用它们。</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">这意味着：只要你的 AI 应用连接了一个 MCP Server，这个 Server 能做的事情，AI 原则上都能做。</p><h2 class="text-2xl font-black text-[var(--color-text)] mt-12 mb-4 tracking-tight border-b border-[var(--color-border)] pb-3">盲区一：工具权限没有细分</h2><p class="text-[var(--color-text-2)] leading-relaxed my-4">MCP 规范里，「工具」是一个原子调用单元。但规范没有定义<strong class='text-[var(--color-text)] font-semibold'>权限层级</strong>。</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">以 GitHub MCP Server 为例，它暴露了几十个工具，包括：</p><ul class='space-y-2 my-5 ml-4 list-disc'><li class="text-[var(--color-text-2)] leading-relaxed">`github_create_repository` — 创建新仓库</li><li class="text-[var(--color-text-2)] leading-relaxed">`github_delete_repository` — 删除仓库</li><li class="text-[var(--color-text-2)] leading-relaxed">`github_add_comment` — 在 Issue 下留言</li></ul><p class="text-[var(--color-text-2)] leading-relaxed my-4">现实场景中，你可能只想让 AI「查询代码库状态」和「评论 Issue」，但你无法阻止它执行 <code class='bg-[var(--color-surface)] text-[var(--color-accent)] px-1.5 py-0.5 rounded text-sm font-mono border border-[var(--color-border)]'>delete_repository</code>。一旦 AI 被提示注入（Prompt Injection）或 MCP Server 本身被恶意改装，删除操作随时可触发。</p><p class="text-[var(--color-text-2)] leading-relaxed my-4"><code class='bg-[var(--color-surface)] text-[var(--color-accent)] px-1.5 py-0.5 rounded text-sm font-mono border border-[var(--color-border)]'>`</code>json</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">// GitHub MCP Server 暴露的部分工具（示意）</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">{</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">"tools": [</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">{"name": "github_list_repos", "description": "List repositories"},</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">{"name": "github_create_repository", "description": "Create a new repo"},</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">{"name": "github_delete_repository", "description": "Delete a repo", "dangerous": true},</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">{"name": "github_add_comment", "description": "Add issue comment"}</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">]</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">}</p><p class="text-[var(--color-text-2)] leading-relaxed my-4"><code class='bg-[var(--color-surface)] text-[var(--color-accent)] px-1.5 py-0.5 rounded text-sm font-mono border border-[var(--color-border)]'>`</code></p><p class="text-[var(--color-text-2)] leading-relaxed my-4">注意那个 <code class='bg-[var(--color-surface)] text-[var(--color-accent)] px-1.5 py-0.5 rounded text-sm font-mono border border-[var(--color-border)]'>dangerous: true</code>——这是 MCP 规范里<strong class='text-[var(--color-text)] font-semibold'>不存在的字段</strong>。目前只是一个社区约定的标记，没有任何协议层面的强制执行机制。</p><h2 class="text-2xl font-black text-[var(--color-text)] mt-12 mb-4 tracking-tight border-b border-[var(--color-border)] pb-3">盲区二：资源 URI 的跨服务逃逸</h2><p class="text-[var(--color-text-2)] leading-relaxed my-4">MCP Resources 通过 URI 标识，比如 <code class='bg-[var(--color-surface)] text-[var(--color-accent)] px-1.5 py-0.5 rounded text-sm font-mono border border-[var(--color-border)]'>file://project/src/app.py</code> 或 <code class='bg-[var(--color-surface)] text-[var(--color-accent)] px-1.5 py-0.5 rounded text-sm font-mono border border-[var(--color-border)]'>postgres://db/users</code>。但 MCP Server 之间<strong class='text-[var(--color-text)] font-semibold'>没有 URI 命名空间隔离</strong>。</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">这导致了一个微妙的攻击向量：</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">1. 你连接了一个 <code class='bg-[var(--color-surface)] text-[var(--color-accent)] px-1.5 py-0.5 rounded text-sm font-mono border border-[var(--color-border)]'>filesystem-mcp</code>（访问本地文件）</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">2. 你还连接了一个 <code class='bg-[var(--color-surface)] text-[var(--color-accent)] px-1.5 py-0.5 rounded text-sm font-mono border border-[var(--color-border)]'>postgres-mcp</code>（访问数据库）</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">3. 恶意提示词可以让 AI 通过 <code class='bg-[var(--color-surface)] text-[var(--color-accent)] px-1.5 py-0.5 rounded text-sm font-mono border border-[var(--color-border)]'>file://</code> URI 直接读取 <code class='bg-[var(--color-surface)] text-[var(--color-accent)] px-1.5 py-0.5 rounded text-sm font-mono border border-[var(--color-border)]'>postgres-mcp</code> 存储的连接凭证</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">更准确地说，不是 MCP 协议本身有这个漏洞，而是<strong class='text-[var(--color-text)] font-semibold'>应用层没有做 URI 路由隔离</strong>。但大多数 MCP Client 实现把这个责任交给了 AI 的推理能力——而 LLM 并不擅长这件事。</p><h2 class="text-2xl font-black text-[var(--color-text)] mt-12 mb-4 tracking-tight border-b border-[var(--color-border)] pb-3">盲区三：MCP Server 来源信任</h2><p class="text-[var(--color-text-2)] leading-relaxed my-4">当你安装一个 npm 包 <code class='bg-[var(--color-surface)] text-[var(--color-accent)] px-1.5 py-0.5 rounded text-sm font-mono border malicious-mcp-server</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">→ 等待开发者 search npm</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">→ 名称近似官方: @modelcontextprotocol/server-github1</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">→ 安装并启用</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">→ Server 可以访问所有已授权工具</p><p class="text-[var(--color-text-2)] leading-relaxed my-4"><code class='bg-[var(--color-surface)] text-[var(--color-accent)] px-1.5 py-0.5 rounded text-sm font-mono border border-[var(--color-border)]'>`</code></p><p class="text-[var(--color-text-2)] leading-relaxed my-4">MCP 官方维护了一个 [Servers 列表](https://github.com/modelcontextprotocol/servers)，但它没有任何代码签名或校验机制。你安装的是谁签名的包？</p><h2 class="text-2xl font-black text-[var(--color-text)] mt-12 mb-4 tracking-tight border-b border-[var(--color-border)] pb-3">实战：攻击链演示</h2><p class="text-[var(--color-text-2)] leading-relaxed my-4">以下是我们内部红队测试的一个概念验证（已脱敏）：</p><p class="text-[var(--color-text-2)] leading-relaxed my-4"><strong class='text-[var(--color-text)] font-semibold'>场景</strong>：Claude Desktop + GitHub MCP + 自定义脚本 MCP</p><p class="text-[var(--color-text-2)] leading-relaxed my-4"><strong class='text-[var(--color-text)] font-semibold'>攻击路径</strong>：</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">1. 攻击者向你的代码库提交一个看似正常的 PR，里面有一个被巧妙构造的 <code class='bg-[var(--color-surface)] // BLOCKED</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">];</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">const mcpClient = new MCPClient({</p><p class="text-[var(--color-text-2)] leading-relaxed my-4">});</p><p class="text-[var(--color-text-2)] leading-relaxed my-4"><code class='bg-[var(--color-surface)] text-[var(--color-accent)] px-1.5
